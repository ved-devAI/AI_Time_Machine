# M4.5 — Real Repo Mode

Status: Required core complete and verified on July 16, 2026. The optional
Codex question artifact is deferred to protect the M5 submission timebox.

## Objective

Prove that AI Time Machine is a developer tool, not only a curated OrbitCart
experience. A developer must be able to point the local application at an
ordinary Git repository and receive an evidence-preserving timeline without an
OpenAI Platform API key.

M4.5 is capped at one focused build session. The existing OrbitCart demo,
deployment, artifacts, and submission reliability must remain unchanged.

## Required developer workflow

Target interface:

```bash
python3 -m app.cli analyze /path/to/repository
python3 -m app.cli serve /path/to/repository
python3 -m app.cli context /path/to/repository --base main --head HEAD
```

Exact command names may change during implementation, but the completed flow
must support:

1. Selecting a local Git repository and branch without GitHub OAuth.
2. Extracting real commits, dates, authors, subjects, changed files, and
   file-overlap history.
3. Producing the existing normalized timeline contract for the browser UI.
4. Serving the selected repository through the current local interface while
   keeping OrbitCart as the deterministic default.
5. Producing change context for a branch or commit range: changed files, recent
   commits touching those files, connected incidents or fixes, and recorded
   risks.

## Evidence behavior for ordinary repositories

Generic repositories will not contain OrbitCart's structured `Summary:`,
`Why:`, and `Risk:` commit-body fields. Real Repo Mode must therefore:

- Preserve the commit subject and diff-derived facts as confirmed evidence.
- Label absent rationale as `not recorded`; never invent a reason.
- Treat classifications derived from conventional-commit prefixes as inferred
  metadata unless the commit explicitly records the type.
- Cite only event IDs, commit hashes, and files extracted from the selected
  repository.
- Keep repository text untrusted when it is supplied to Codex.

## Proof repository

AI Time Machine must analyze its own Git history as the second proof case.
OrbitCart remains the polished incident narrative; the self-analysis proves
that ingestion and rendering are not hardcoded to that narrative.

The demo proof should show at least:

- The M0–M4 milestone commits in chronological order.
- Files connected across multiple milestones.
- A branch or `main..HEAD` context report grounded in real changed files.
- Clear `not recorded` labels wherever Git does not contain rationale.

## Optional Codex extension

If the required workflow is complete and verified, add an opt-in command that
exports a compact evidence bundle and strict schema for a developer question.
The answer may be generated with ChatGPT-authenticated `codex exec`, then
validated and replayed locally using the existing artifact pattern.

This extension must not require a Platform API key. Output must identify
whether it came from a validated Codex artifact or the local evidence engine.
Free-form, unvalidated chat is out of scope.

## Acceptance criteria

- Works on a temporary test repository that has no structured commit bodies.
- Works when pointed at the AI Time Machine repository itself.
- Requires only Python 3.11+ and Git for the required workflow.
- Does not expose arbitrary filesystem paths through the public deployment.
- Does not weaken event, commit, file, certainty, or provenance validation.
- Keeps the public OrbitCart demo and `python3 scripts/verify.py` passing.
- Adds tests for invalid paths, non-Git directories, empty history, branch/range
  errors, and rationale that is not recorded.
- Updates `CONTEXT.md`, `ROADMAP.md`, `README.md`, and the decision log after the
  implementation is verified.

## Explicit non-goals

- GitHub OAuth or private-repository hosting
- A GitHub App or automated pull-request comments
- VS Code or JetBrains extensions
- Team accounts, persistent indexing, or cloud repository storage
- Multiple source-control providers
- Autonomous code modification
- Replacing the three-minute OrbitCart flagship flow

## Stop condition

If generic ingestion, self-analysis, and the local developer workflow are not
solid within the timebox, ship those pieces without the optional Codex command.
M5 demo and submission work takes priority over additional integrations.
